CRUD - Laravel

Instruções
### Atividade: Sistema de Gerenciamento de Produtos com Laravel 

 

#### Objetivo: 

Construir um aplicativo Laravel para gerenciar um catálogo de produtos de um estoque. 

 

#### Requisitos Funcionais: 

 

1. **Modelo de Dados**:  

   - Crie um modelo `Products` com os seguintes atributos: 

     - `name` (string) 

     - `producer` (string) 

     - `code` (string, único) 

     - `production_date` (integer) 

     - `quantity` (integer) 

 

2. **Migrations**:  

   - Crie uma migration para a tabela ` Products ` com os campos acima. 

 

3. **Seeders**: 

   - Crie um seeder para preencher a tabela ` Products ` com dados fictícios. 

 

4. **Rotas**: 

   - Defina rotas para as operações CRUD. 

 

5. **Controllers**: 

   - Crie um controlador ` ProductController` com métodos para as operações CRUD. 

 

6. **Views**: 

   - Crie views para as seguintes funcionalidades: 

     - Listar todos os produtos (`index`). 

     - Exibir um formulário para adicionar um novo produto (`create`). 

     - Exibir um formulário para editar um produto existente (`edit`). 

     - Exibir detalhes de um produto (`show`). 

 

7. **Validação**: 

   - Implemente validação nos métodos `store` e `update` do controlador. 

 

8. **Testes**: 

   - (Opcional) Escreva testes básicos para garantir que o CRUD está funcionando como esperado. 

 

#### Passos Recomendados: 

 

1. **Configuração Inicial**:  

   - Crie um novo projeto Laravel. 

   - Configure seu ambiente de desenvolvimento e banco de dados. 

 

2. **Desenvolvimento**: 

   - Siga os requisitos funcionais para desenvolver cada parte do CRUD. 

   - Teste cada funcionalidade à medida que for desenvolvendo. 

 

3. **Estilização**: 

   - Utilize Bootstrap ou outro framework CSS de sua escolha para estilizar as views. 

 

4. **Refinamento**: 

   - Revise o código para garantir boas práticas. 

   - Refatore o código para melhorar a eficiência e legibilidade. 

 

5. **Documentação**: 

   - Documente o processo de instalação e configuração do aplicativo. 

   - Escreva um README claro e conciso.
